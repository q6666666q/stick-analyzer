## 🆕 v2.1 更新亮点（2026-05）

### 新分析维度
- **腰射 vs 开镜 不对称分析**（第七节）：差异 > 30% 自动给针对性曲线建议——告诉你具体调 ADS 曲线还是腰射曲线
- **走位 vs 站桩 模式对比**（第八节）：走位组用 ×1.3 阈值放宽（左右手协调更难），区分"曲线问题"和"走位本身的劣化"
- **过冲细分类型**（第三节）：
  - 大幅过冲（>0.15 幅度，甩过头）→ 调高段曲线
  - 小抖动（0.05-0.15 幅度，微修正）→ 调低段曲线 / 检查死区 / RC 强度
- **行为分类细化**（第四节）：从 4 档扩展到 8 档（完美稳定 ⭐ → 稳定射击 ✓ → 接近稳定 → 中等稳定 → 微调跟枪 → 三种问题档），每档配玩家直觉解释

### TMR 认知更新
- TMR 算法已成熟，延迟和线性度接近碳膜 ALPS（主流 FPS 手柄出厂默认）
- 不再把 TMR 跟霍尔放在一起做「中心钝化」处理——TMR 阈值跟碳膜同档（×1.0）
- 反死区补偿建议**只对霍尔触发**（霍尔仍是非 FPS 主流）

### 关键 bug 修复
- **SDL 跨线程 button 失效**：修复了部分手柄（尤其第三方 XBOX 360 兼容控制器）

- **8000Hz 高回报率手柄卡顿**：屏蔽 SDL joystick events，事件队列保持空

### AI 提示词增强

## 📖 这个工具是干嘛的（30 秒看懂）

> **简单说**： 你正常打游戏 → 软件录下你摇杆的每一次动作 → 自动分析你"瞄得稳不稳、压得稳不稳、有没有甩过头" → 生成一份带图带数据的报告 → 顺便给你一段提示词，丢给 AI 让 AI 帮你算曲线。
>
> 整个过程**全程不操作游戏、不发虚拟输入、反作弊看不见**，只是事后分析你自己的手柄数据。

### 适合谁

- ✅ FPS 手柄玩家（Apex / 战地 / TheFinals / COD 等）
- ✅ 已经在用调参 APP（北通 / 飞智 / 莱仕达 / 八位堂等）改过曲线，**但不确定改得对不对**的人
- ✅ 想知道自己**压枪到底稳在哪、不稳在哪**，而不是凭感觉的人
- ✅ 想把"我压不住"翻译成具体数字（比如"压枪稳定度 0.116，开镜段比腰射段抖 30%"）的人

### 不适合谁

- ❌ 鼠标键盘玩家（这工具只分析摇杆数据）
- ❌ 想用工具检测手柄硬件是否损坏的人（这不是测试仪，硬件检测请用 gamepadtester 之类的工具）
- ❌ 期望工具直接告诉你"按这个曲线打"的人——工具给的是数据 + AI 协作的输入，**最终曲线还是要 AI 算 + 你自己调**

### 能解决什么问题

- "我压枪感觉飘但不知道是手抖还是曲线问题" → 数据告诉你
- "我换了曲线感觉没区别" → 数据对比看有没有真改善
- "我想知道开镜和腰射哪个更不稳" → 报告里直接分两组对比
- "AI 让我改 X=33-41 段曲线，但我不知道为啥" → 报告里有"主导推杆区间"指标解释清楚

### 不能解决什么问题

- 不能替你打游戏（不会自动压枪）
- 不能给你完全针对你的"最优曲线"——曲线没有最优，只有"匹配你目前手感"的
- 不能在 PVP 实战里给你绝对干净的数据（变量太多，建议用靶场数据）

```
① 录制      → ② 分析       → ③ 调参（AI 协作）
打靶/打游戏     看数据和波形    给 AI 看报告，AI 给出曲线建议
```

**典型用法举例**：

> 进 Apex 训练场，挑一把武器（比如 R-99）→ 启动本工具点录制 → 软件先让你松手 3 秒做硬件校准 → 然后正常打靶 30-60 秒 → 停止录制 → 分析 → 看报告里你**主导推杆区间在 X=33-41 段**且**这段抖动最大** → 把报告丢给 AI → AI 告诉你**把 X=33-41 段对应曲线节点 Y 值降低 0.5-1 单位** → 进调参 APP 改完 → 再录一次对比是否改善。


```bash
pip install pyinstaller
python build_exe.py
# 选 1（onedir 推荐）或 2（onefile）
```

输出在 `dist/` 目录。

### 核心指标

| 指标             | 含义                                      | 优秀                         | 良好  | 一般  | 差    |
| ---------------- | ----------------------------------------- | ---------------------------- | ----- | ----- | ----- |
| **开火前稳定度** | 扣扳机前 100ms 准星稳不稳                 | <0.04                        | <0.06 | <0.10 | >0.10 |
| **开火中稳定度** | 压枪 300ms 过程中（去掉压枪趋势后）抖不抖 | <0.04                        | <0.06 | <0.08 | >0.08 |
| **反转次数**     | 过冲严重程度（甩过头然后回拉）            | <8                           | <12   | <20   | >20   |
| **大幅过冲** 🆕  | 单次反转 > 0.15 幅度（甩过头）            | 反映高段曲线斜率             |       |       |       |

## 🆕 录制前会有 3 秒静止校准

这一步不是 bug，是新增的精度提升机制：

- 软件让你**松开摇杆 3 秒**，记录摇杆完全静止时的微小波动（"传感器本底"）
- 这个本底来自硬件本身（回中虚位、电气噪声，霍尔摇杆尤其常见）
- 后续报告里会**自动减掉这部分**，让你看到的"压枪不稳"是真正手部+曲线引起的，不混入硬件噪声
- 如果你的本底数值偏高（>0.015），报告会提示可能是回中弹簧老化、漂移迹象、或周围磁场干扰

## 🆕 实际有效采样率检测

软件采样上限由 pygame/SDL 协议决定（通常 500-1000Hz），**与你手柄的回报率高低无关**——即使是 4000Hz / 8000Hz 高端手柄，软件这层也只能取到 SDL 协议范围内的数据。

- 这不是 bug，是协议限制；对压枪分析（关注 5-50Hz 频段）完全够用
- 录制时自动统计重复帧比例
- 实际有效采样率 < 200Hz 才会警告（蓝牙断连 / CPU 占用过高 / 第三方驱动限频等真问题）

## 🔧 常见问题

### Q：录制时 Fire=0%，按键也没反应？

A：检查录制时的开火键设置是否对。用扳机当开火的话选「右扳机 (RT)」/「R2 右扳机」。

### Q：槽位列表没显示我的手柄？

- 确认手柄已连接（USB 插稳 / 蓝牙配对完成）
- 点「🔄 刷新设备列表」
- 部分手柄需要先在 Windows 设备管理器里识别为「人体学输入设备」

### Q：为什么启动时窗口要等几秒才出来？

A：v2.0+ 起窗口立即弹出，手柄驱动初始化在后台异步进行（看到"正在初始化手柄驱动…"提示是正常的）。如果你的版本仍然要等几秒才弹窗，请更新到最新版。

### Q：录制时只有扳机能识别，按键全失效？

A：v2.1 已修复。这是 SDL 在某些第三方手柄（尤其山寨 XBOX 360 兼容芯片）上的跨线程 bug，新版自动启用 SDL 独立 joystick 线程绕开。如果仍有问题，确认你用的是 v2.1+ 版本。

### Q：报告说我"有效采样率 < 250Hz"是什么意思？

A：你的手柄到电脑的实际数据更新率较低。常见原因：

- 蓝牙连接（建议改有线 USB）
- USB 端口是低速 USB 1.1（换 USB 2.0+ 端口）
- 第三方驱动限制了报告率

> 这种情况下分析结果会偏乐观（少报抖动），建议有线连接重测。

### Q：报告说"传感器本底偏高"，是手柄坏了吗？

A：不一定。可能性：

- 霍尔/TMR 摇杆的中心特性（中心钝化区天然有微小波动）
- 回中弹簧老化（机械原因，所有传感器类型都会发生）
- 摇杆周围有强磁场（手机、风扇、显示器）
- 漂移早期迹象

> 报告里会自动减掉这部分，不影响分析结果，但提示你关注硬件状态。

### Q：生成的图中文乱码？

A：确保系统装了 Microsoft YaHei 字体（Windows 自带）。

### Q：EXE 被杀软误报？

A：PyInstaller 打包的 EXE 经常被误判，加入白名单或自己用源码打包即可。

### Q：CSV 文件太大？

A：1 小时记录约 200-300MB，长时间游戏建议分段记录。

### Q：任务管理器里看到两个 StickAnalyzer.exe 进程？

A：如果你用的是 onefile 单文件版，这是 PyInstaller bootloader 的正常机制（一个负责解压临时文件，一个是真正的程序），不是 bug。如果想避免，用 zip 版（onedir 模式）只有一个进程。

## 📝 设计原则

工具基于这些原则设计：

1. **数据辅助，体感为主** — 数据用来发现具体问题，最终判断标准始终是体感
2. **不假设硬件** — 不同手柄的曲线点数、RC 范围、按键映射都不同，工具自适应
3. **不引导过度调参** — 频繁切换曲线会污染肌肉记忆，工具会提醒
4. **绝对安全** — 永远不主动向游戏发送任何输入，所有调整都由玩家手动完成
5. **AI 协作** — 工具是数据收集 + 整理者，AI 是综合分析 + 建议者，玩家是最终决策者
