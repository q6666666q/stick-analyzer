## ✨ v2.0 主要功能

- **双驱动控制器支持** — pygame（PS4/PS5/DualSense Edge/通用）+ XInput（XBOX）
- **4 槽位手柄管理** — 同时识别多个手柄，按插入顺序占位
- **按键标签自动适配** — PS 手柄显示「× ○ □ △」，XBOX 显示「A B X Y」
- **DualSense Edge 背键支持** — FN1 / FN2 / RB1 / RB2 可识别和录制
- **500Hz 高精度记录** — 实时记录手柄摇杆位置、按键状态、扳机值
- **自动事件检测** — 无需外部时间戳，自动识别每次开火爆发
- **多维度分析** — 开火前稳定度、开火中稳定度、过冲次数、推杆量分布
- **可视化报告** — 每次开火生成波形图 + 总览统计图 + 文字报告
- **AI 调参提示词生成** — 自动包含分析数据，丢给 AI 即可获得针对性建议
- **参考曲线收集指南** — 引导玩家从社区收集参考曲线，让 AI 综合判断
- **GUI 友好** — 图形化界面，键位下拉选择，无需懂代码
- **绝对安全** — 不操作游戏、不发送虚拟输入、反作弊系统看不到任何异常

## 🚀 完整使用流程（4 个标签页）

### ① 录制摇杆数据

1. 选择控制器：从 4 个槽位中选定要录制的手柄
   - 自动识别 PS / XBOX / Switch / 通用 HID 等所有手柄
   - 同一个手柄被两个驱动识别时优先用 pygame
2. 选择开火键和开镜键：下拉框自动显示该手柄的原生标签
3. 填写元数据（RC 数值、动感强度等级、武器、场景等）
   - 不同手柄 RC 范围不同（±10 / ±100 / ±500），按比例选择强度等级
4. 点「● 开始录制」→ 正常打游戏 → 点「■ 停止录制」

### ② 分析数据

1. CSV 路径自动填入或手动选择
2. 点「▶ 开始分析」
3. 输出文件（与 CSV 同目录）：
   - `*_report.txt` — 文字分析报告
   - `*_event_NN.png` — 每次开火的波形图
   - `*_summary.png` — 总览统计图

### ③ 生成 AI 调参提示词

1. 自动嵌入录制分析报告
2. 玩家在文本框中填入：
   - 手柄型号、曲线编辑方式、可调节点数
   - 当前曲线坐标
   - 体感痛点
3. 点「📋 复制全部到剪贴板」
4. 粘贴给 AI（推荐 Claude / ChatGPT）

### ④ 参考曲线收集

1. 看左侧指南，了解从哪找别人调好的参考曲线
   - B站 / 抖音 / 贴吧 / Reddit / Discord
   - 调参 APP 内置的社区配置
   - 游戏内截图给 AI 识别
2. 把收集到的 3-5 条参考曲线粘贴到右侧文本框
3. 点「➕ 注入到 AI 提示词」自动跳到标签 ③
4. 提示词里现在包含「录制分析 + 参考曲线 + 待填的痛点」
5. 复制完整内容给 AI → AI 综合判断给出个性化建议

## 📊 分析指标说明

| 指标 | 含义 | 优秀 | 良好 | 一般 | 差 |
|------|------|------|------|------|----|
| 开火前稳定度 | 瞄准时准星稳不稳 | <0.04 | <0.06 | <0.10 | >0.10 |
| 开火中稳定度 | 压枪过程稳不稳 | <0.04 | <0.06 | <0.08 | >0.08 |
| 反转次数 | 过冲严重程度 | <8 | <12 | <20 | >20 |

> 🔧 阈值会根据 RC 动感强度自动调整。

## 🔧 常见问题

### `Fire=0%` 即使按键也没反应？
检查录制时的开火键设置是否对。如果用的是扳机当开火，选「右扳机 (RT)」/「R2 右扳机」。

### 槽位列表没显示我的手柄？
- 确认手柄已连接（USB 插稳 / 蓝牙配对完成）
- 点「🔄 刷新设备列表」
- 部分手柄需要先在 Windows 设备管理器里识别为「人体学输入设备」

### 生成的图中文乱码？
确保系统装了 Microsoft YaHei 字体（Windows 自带）。

### EXE 被杀软误报？
PyInstaller 打包的 EXE 经常被误判，加入白名单或自己用源码打包即可。

### CSV 文件太大？
1 小时记录约 200-300MB，长时间游戏建议分段记录。

## 📝 设计原则

本工具基于这些核心原则设计：

1. **数据辅助，体感为主** — 数据用来发现具体问题，最终判断标准始终是体感
2. **不假设硬件** — 不同手柄的曲线点数、RC 范围、按键映射都不同，工具自适应
3. **不引导过度调参** — 频繁切换曲线会污染肌肉记忆，工具会提醒用户
4. **绝对安全** — 永远不主动向游戏发送任何输入，所有调整都由玩家手动完成
5. **AI 协作** — 工具是数据收集 + 整理者，AI 是综合分析 + 建议者，玩家是最终决策者
